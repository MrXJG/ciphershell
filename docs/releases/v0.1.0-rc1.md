# GMSSH Client v0.1.0-rc1

Release type: internal release candidate.

## Scope

- Qt/C++ desktop GM SSH client.
- macOS arm64 app bundle / DMG.
- Windows x64 NSIS installer and portable zip.
- Bundled modern GM OpenSSH engine and legacy EC-GM compatibility engine.

## Verified Capabilities

- Standard SSH connection mode.
- openEuler GM mode using `sm2-sm3`.
- Kylin V10 SP3 / 2403 legacy GM mode using `ecgm-sm2-sm3`.
- Windows GUI launch after install.
- Windows desktop shortcut creation.
- Windows install-directory audit log creation: `log/audit.log`.
- Windows dependency closure for GUI and SSH engines.
- Windows modern and legacy engine algorithm availability.
- SFTP engine fallback and packaged engine wiring.
- Terminal audit logging for input/output events.

## Known Limitations

- Windows terminal backend currently uses a Qt `QProcess` pipe mode, not a full ConPTY backend. Normal SSH shell use is supported, but full-screen TUI programs may need a future ConPTY implementation.
- GM host-signature legacy adaptation reduces verification strength and should only be enabled for old EC-GM servers that require it.
- Release candidate is intended for internal validation before a final `v0.1.0` tag.
